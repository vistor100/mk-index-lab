# mk-index-lab

## 项目简介

mk-index-lab 是一个用于整理、归档和发布多个独立 HTML 页面的仓库。  
本仓库本身不绑定任何特定域名或网站，主要用于存放各类静态页面资源，便于版本管理与分发。

## 目录结构说明

```
.
├── index.html          # 入口页面，展示所有归档页面链接
├── pages/              # 存放各个独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 文档或说明文件
└── README.md           # 本文件
```

- `pages/` 目录下每个 HTML 文件为一个独立归档页面。  
- `assets/` 中存放页面共用的样式、脚本与图片资源。  
- `docs/` 可放置额外的使用说明或开发笔记。  
- `index.html` 提供页面索引，方便快速浏览。

## 页面归档说明

- 每个 HTML 文件应保持独立，不依赖外部不可控资源（建议将资源放置于 `assets/` 目录内）。  
- 页面内容应结构清晰，不含敏感或违规信息。  
- 归档页面可包含演示、工具、信息展示等用途，但不应包含商业推广或诱导点击内容。  
- 如需新增页面，请在 `pages/` 下创建新文件，并在 `index.html` 中添加对应链接。

## 维护说明

- 本仓库由维护者不定期更新。  
- 欢迎提交 Issue 或 Pull Request 进行改进或新增页面。  
- 提交前请确保页面内容符合开源社区规范，不包含恶意代码或违法信息。  
- 本仓库仅作为静态页面托管与索引，不对页面内容的准确性或时效性作保证。
